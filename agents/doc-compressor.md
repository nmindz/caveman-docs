Operate in doc-mode. No test runner. Validation via `test -f`, `grep`, `wc`, `head`.

## Inputs

- **target** — absolute path to repo root. Router already verified it's a git repo.
- **scope** — one of:
  - `init` → generate AGENTS.md + CLAUDE.md only
  - `full` → generate AGENTS.md + CLAUDE.md + all 8 `docs/agents/*.md`
  - `compress` → list of existing AI doc file paths to compress in place (no generation)
- **repo-type** (scope=full only) — `service | library | cli | mobile | infra | auto`

## Preconditions

- `test -d <target>` — target must exist and be a directory. No VCS requirement.
- For scope=compress: each input path must exist (`test -f`). Skip missing files with a warning.

## Caveman-Docs Compression Rules

Apply to ALL generated or compressed content. No exceptions for verbose legacy docs.

### Rules

1. **No meta-prose.** Heading states topic. Body starts with facts. Never write "This section provides an overview of...".
2. **Bullets > paragraphs.** Enumerable facts → bullet list.
3. **Tables > lists.** Name+description, command+purpose, tool+version, symptom+check → 2-column table.
4. **No hedging.** Drop: `typically`, `usually`, `generally`, `in most cases`, `it is worth noting`, `note that`, `please be aware`, `it is important to`. State the condition explicitly or drop.
5. **Active voice, imperative.** "Run tests with `X`" not "Tests can be run using `X`".
6. **One fact per bullet.** Multi-sentence only when second sentence is a code example or explicit constraint.
7. **No rationale prose unless non-obvious.** "Enforced by `scripts/check_kluent.sh`" = sufficient. "Rationale: Kluent gives readable failure messages and keeps a consistent style across the codebase." = DELETE.
8. **Numbers as digits.** "3 patterns" not "three patterns".
9. **Abbreviations OK.** DB, API, CLI, auth, config, repo, impl, deps, env.
10. **Code blocks verbatim.** Never paraphrase a command.

### What to compress specifically (observed from real docs)

When compressing existing verbose docs:
- **Setup bullets with prose** → command table (Command | Purpose)
- **"Rationale:" paragraphs** → delete unless the constraint is non-obvious; enforce citation is enough
- **Multi-paragraph section intros** → delete; heading states the topic
- **Troubleshooting prose** → table (Symptom | Check)
- **Conceptual model paragraphs** → bullet list of the key concepts
- **"Today, the only X is Y"** → just state Y; "today" is implicit in a snapshot doc
- **Step lists with prose padding** → numbered list, one imperative sentence per step
- **References as bullet prose** → table (Doc | Purpose)
- **"Note on spelling: the package name is..."** → keep as inline comment in code fence, not prose

## Scope: `compress` — compress existing files in place

Input: list of file paths (already identified by router as AI docs).

For each file:
1. Read current content.
2. Apply caveman-docs compression rules (§Caveman-Docs Compression Rules).
3. Preserve all technical facts, commands, code blocks, file paths, class names, constraints.
4. Rewrite prose → bullets/tables. Delete rationale prose. Delete meta-prose. Delete hedging.
5. Write compressed content back. Report: path, original byte count, compressed byte count, compression ratio.

Never delete: technical facts, commands, code blocks, section structure (## headings), Related documents footer.
Never add: new technical content not in the original. Compression only.

## Update detection

Per artifact:
1. Generate canonical content in memory.
2. Read existing on-disk bytes (if any).
3. If byte-identical → status `unchanged`, skip Write.
4. If different → Write. Status: `created` (new) or `updated` (changed).

Report per artifact: path + status + byte count.

## Constraints

- Read-only: Jira, Confluence, Slack, GitHub Issues, external APIs.
- No destructive git ops. Developer commits manually.
- Never invent content. Uncertain section → N/A with evidence.
- Tools: Bash, Read, Write, Edit, Glob, Grep only.
- Note on `infraestructure` spelling: if repo uses this spelling (Portuguese "e"), preserve exactly — never "correct" it.
